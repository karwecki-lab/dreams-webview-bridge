💡 PRZYKŁADY UŻYCIA

Swift → React:
  let message = [
    "type": "greeting",
    "payload": ["text": "Hi from Swift!"]
  ]
  coordinator.sendMessageToJavaScript(message)

React → Swift:
  window.webkit.messageHandlers.nativeApp.postMessage({
    type: 'greeting',
    payload: { text: 'Hi from React!' }
  })

Więcej: README.md sekcja "Examples"

🔍 ARCHITEKTURA

iOS (MVVM):
  ContentView (UI)
    ↓
  WebViewViewModel (Logic)
    ↓
  WebViewContainer (Bridge)
    ↓
  WKWebView + Coordinator

React:
  User Action → postMessage() → Swift
  Swift → evaluateJavaScript() → React State → UI

  → Sprawdź logi w Xcode Console

Problem: Build error
  → Clean: Cmd+Shift+K, potem Build: Cmd+B

📊 PORÓWNANIE: Native vs SDK

Native (ten projekt):
  ✅ Zero dependencies
  ✅ Prosty kod (~200 linii)
  ✅ Szybki build
  ✅ Łatwy debugging
  ✅ Stabilny
  
Z SDK (np. Dreams):
  ✅ Zaawansowana analytics
  ✅ Gotowe integracje
  ⚠️  Dodatkowa zależność
  ⚠️  Więcej kodu
  ⚠️  Dłuższy build

WYBIERZ NATIVE gdy:
  → Chcesz prostoty
  → Budujesz od podstaw
  → Nie potrzebujesz external analytics
  → Wartość stabilności

DODAJ SDK gdy:
  → Potrzebujesz zaawansowanej analytics
  → Masz gotową integrację
  → Zespół już używa SDK
